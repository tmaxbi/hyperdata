[HDDS-1084] Recon server 1 (implemented)

Authors: Sid Wagle, Vivek Subramanian and Aravindan
2019-02-19

 

Summary
Second phase of Recon development. Recon acts as a passive SCM.

Abstract

Recon is a monitoring and analytics server for Ozone. It can retrieve data from other components and store it and do historical queries (eg. out-of-disk-space time estimation).

Ozone Recon (short for reconnaissance) Service is, as the name suggests, meant to be eyes and ears of Ozone for the user. The first resort in understanding what is happening with Ozone.
